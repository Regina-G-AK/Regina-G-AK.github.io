一、逻辑处理

1.1 CASE WHEN

  1. 查询函数形式：在 WHEN 后面写任何复杂的逻辑表达式（比如比较大小、使用 AND/OR、模糊查询等）。

Listing 1: CASE WHEN查询示例

  1. 简单函数形式：主要用于等值判断，即判断某个字段是否等于某个具体的值。

Listing 2: CASE WHEN函数示例

注意：CASE WHEN 是从上到下依次判断的。一旦满足了某个条件，就会返回对应的结果，不再看后面的条件；THEN 后面返回的所有值，数据类型必须是一致的。

二、窗口函数

  1. 移动平均+滚动聚合：ROWS BETWEEN 6 PRECEDING AND CURRENT ROW窗口范围限定了计算范围；DENSE_RANK可以过滤掉前6条数据。

Listing 4: 餐厅7天营业额，leetcode1321

  1. 多维条件判断：使用多种属性进行逻辑判断时，可以使用窗口函数。

Listing 5: 多属性条件判断，leetcode585

  1. Top-N问题：不能在定义窗口函数的同层查询中使用它的别名进行过滤。在Top-N问题中，必须套一层子查询，先算出排名，再在外面过滤。

Listing 6: 部门工资前三高的员工，leetcode185

三、双向关联

类似“好友关系”、“比赛对阵”、“转账双方”这种数据分布在两列、且身份可以互换的场景时：

  1. 用 SELECT A FROM table UNION ALL SELECT B FROM table 将两列化为一列。

  2. 对外层结果进行 GROUP BY 和 COUNT。

注意：使用UNION ALL可以保留含有重复的列，适用于双向关联的问题。

Listing 7: 拥有好友数量最多的人，leetcode602

四、字符串函数

  1. 转换大小写：LOWER()将字符串转换成小写，UPPER()将字符串转换成大写。

  2. 字符串连接：CONCAT()将两个/多个字符串拼接。

  3. 纵向合并多行至一行：GROUP_CONCAT(feature, SEPARATOR c)将多个特征值拼接到一起，并且采用字符c作为分隔符。

Listing 8: 多个特征值拼接，leetcode1484

  1. 去空格：LTRIM()将字符串左侧空格去除，RTRIM()将字符串右侧空格去除。

  2. 取子串：SUBSTRING(<expression>, <starting_position>, length)返回从字符串左边第starting_position 个字符起length个字符的部分。

  3. 重复：REPLICATE(<expression>, n)返回一个重复expression指定次数n的字符串。

  4. 替换：REPLACE(<string_expression1>, <string_expression2>, <string_expression3>) 用string_expression3 替换在string_expression1 中的子串string_expression2。

  5. 翻转：将指定的字符串的字符排列顺序颠倒。REVERSE(<character_expression>)其中character_expression可以是字符串、常数或一个列的值。

五、正则表达式

Listing 9: 开头正则匹配，leetcode1527

Listing 10: 查找有效邮箱正则匹配，leetcode1517

正则表达式 解释
\d 匹配任意数字（等价于 [0-9]）
\D 匹配任意非数字字符（等价于 [^0-9]）
\w 匹配任意字母、数字或下划线（等价于 [a-zA-Z0-9_]）
\W 匹配任意非字母、数字或下划线的字符（等价于 [^a-zA-Z0-9_]）
\s 匹配任意空白字符（如空格、制表符、换行符等）
\S 匹配任意非空白字符
\b 匹配单词边界
\B 匹配非单词边界
\n 匹配换行符
\r 匹配回车符
\t 匹配制表符
. 匹配任意单个字符（除换行符外）
* 匹配前面的字符或子表达式零次或多次
+ 匹配前面的字符或子表达式一次或多次
? 匹配前面的字符或子表达式零次或一次
{n} 匹配前面的字符或子表达式恰好 n 次
{n,} 匹配前面的字符或子表达式至少 n 次
{n,m} 匹配前面的字符或子表达式至少 n 次，最多 m 次
^ 匹配字符串的开头
$ 匹配字符串的结尾
[abc] 匹配字符集合中的任意一个字符（如 a、b 或 c）
[^abc] 匹配不在字符集合中的任意一个字符
[a-z] 匹配任意一个小写字母
[A-Z] 匹配任意一个大写字母
[0-9] 匹配任意一个数字
\1 匹配第一个捕获组的内容（用于反向引用）
\2 匹配第二个捕获组的内容（用于反向引用）
\n 匹配第 n 个捕获组的内容（用于反向引用）
(pattern) 定义一个捕获组，用于捕获匹配的内容
(?:pattern) 定义一个非捕获组，不保存匹配的内容
| 表示逻辑“或”操作
