大模型基础

Author: Regina Date: 三月 22, 2026  20:33:00 Category: LLM

1. 架构基石：Attention 机制及其演进

注意力机制是 Transformer 的核心，其演进目标主要在于平衡计算效率与表达能力。

1.1 基础注意力公式

$$Attention(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

  • Self-Attention：$Q, K, V$ 均来自同一输入。加入掩码（Mask）后可实现因果推断：$\text{softmax}\left(\frac{MQK^T}{\sqrt{d_k}}\right)V$。

  • Multi-Head Attention (MHA)：将特征投影到多个子空间并行计算，增强模型捕获多维度信息的能力。

1.2 显存与计算优化

随着上下文长度（Context Length）的增加，$O(N^2)$ 的复杂度成为瓶颈。

  • Flash Attention：IO 感知型算法。利用 GPU SRAM 高速缓存进行计算，通过在线 Softmax 技巧避免将大尺寸的注意力矩阵写回显存（HBM），实现 2-4 倍加速。

  • Sparse Attention：只计算局部或特定模式的注意力，将复杂度降为次线性。

  • Linear Attention：利用核函数特性交换计算顺序（先算 $K^TV$），将复杂度降低至 $O(N)$。

1.3 显存节省型变体（KV Cache 优化）

为了减少推理时 KV Cache 的显存占用：

  • Multi-Query Attention (MQA)：所有头共享一组 K、V。显著降低显存开销，但可能略微损失精度。

  • Grouped-Query Attention (GQA)：MHA 与 MQA 的折中方案。将 Q 分组，每组共享一组 K、V，是目前 Llama 3 等主流模型标配。

2. 大规模分布式训练

2.1 数据并行（DP）与 ZeRO 优化

ZeRO (Zero Redundancy Optimizer) 是 DeepSpeed 提出的核心技术，通过消除冗余提升显存利用率：

  • ZeRO-1：对 优化器状态（Optimizer States） 进行分片。

  • ZeRO-2：进一步对 梯度（Gradients） 进行分片。

  • ZeRO-3：最彻底的分片，连 模型参数（Parameters） 也分布在不同卡上。计算时通过 All-Gather动态获取，用完即弃。

扩展：ZeRO-Offload 和 ZeRO-Infinity 将存储延伸至 CPU 内存乃至 NVMe 硬盘，实现小显存跑大模型。

2.2 模型并行（MP）

  • 张量并行 (Tensor Parallelism, TP)：算子级拆分。将大矩阵乘法拆分到多个 GPU。通信频率极高，通常限制在 NVLink 内。以矩阵乘法 $Y = XA$ 为例，将矩阵 $A$ 按列切分为 $A_1$ 和 $A_2$。GPU 0 计算 $XA_1$，GPU 1 计算 $XA_2$，最后通过通信（All-Gather）把结果拼起来。TP 的痛点：通信延迟。

  • 流水线并行 (Pipeline Parallelism, PP)：层级拆分。不同 GPU 负责模型的不同层。假设模型有 100 层，GPU 0 负责前 50 层，GPU 1 负责后 50 层。数据像传送带一样，先在 GPU 0 跑完，再传给 GPU 1。PP 的痛点：流水线气泡。（如果数据只是简单地从 GPU 0 传到 GPU 1，那么当 GPU 1 计算时，GPU 0 就在闲着。）为解决流水线气泡问题，通常结合 Micro-batch 技术提高吞吐。

3.微调与人类偏好对齐 (Alignment)

模型在预训练后拥有了知识，但需要通过对齐才能听懂指令并符合规范。

3.1 监督微调 (SFT)

通过高质量的（Prompt, Response）对，让模型学习特定格式和风格。它是 RLHF 的基石。

3.2 强化学习家族 (RLHF & Beyond)

为了解决 SFT 无法处理的偏好排序问题：

  • PPO (Proximal Policy Optimization)：传统 RLHF 核心。包含 Actor、Reference、Reward、Critic 四个模型，通过剪切机制保证策略平稳更新。

  • DPO (Direct Preference Optimization)：目前的工业界主流。将 RL 问题转化成二元交叉熵损失，不需要训练Reward和Critic模型，训练更稳定高效。

$$L_{DPO} = -\mathbb{E} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right]$$

  • ORPO：单阶段对齐。在 SFT 阶段加入几率比（Odds Ratio）惩罚，直接区分好坏回答。

  • GRPO (Group Relative Policy Optimization)：DeepSeek-R1 采用的核心技术。取消了 Critic 模型，改用群体评分（Group Relative）来计算优势函数，极大节省显存，特别适合逻辑推理任务。

$$Advantage_i=\frac{Score_i-mean(Score)}{std(Score)}$$

4. 采样生成

推理阶段的采样参数直接决定了模型输出的质量和风格。

4.1 温度 (Temperature)

作用于 Softmax 前的 Logits：$P_i \propto \exp(Logit_i / T)$。

$$P_i = \frac{\exp(q_i / T)}{\sum_j \exp(q_j / T)}$$

  • $T \to 0$：趋近于 Greedy Search，输出确定、严谨。

  • $T > 1$：分布变平坦，增加随机性与多样性，适合创作。

4.2 概率过滤

  • Top-k：取概率最高的前 k 个词。

  • Top-p (核采样)：动态选择累积概率达到 $p$ 的最小词集。比 Top-k 更具适应性，因为候选词空间会随模型自信度动态调整。

4.3 搜索算法

  • Greedy Search：每一步选最强词，易陷入局部最优。

  • Beam Search (束搜索)：保留 $k$ 个最有潜力的候选序列，平衡探索与利用。

5. 演进规律：Scaling Laws

  • Kaplan Scaling Laws：早期研究认为参数量（N）对性能提升最明显，即使数据量（D）不够大，也可以靠模型规模硬堆。

  • Chinchilla Scaling Laws：后来的共识认为，模型参数（N）与数据量（D）应等比例增长。黄金准则：每增加 1 个模型参数，需要约 20 个 Token 的训练数据。
